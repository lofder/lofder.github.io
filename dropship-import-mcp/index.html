没解决的那块 / The gap

选品-编辑-推送。Product curation — import, edit, push.

DSers 自己的官方教程就叫 "5 Simple Steps"——找商品、导入、编辑、推送、管理。听起来简单，但每个"step"都是一堆点击：

DSers' own tutorial is called "5 Simple Steps" — find, import, edit, push, manage. Sounds simple. But each "step" is a pile of clicks:

Find a product → click import → wait → edit title (remove Chinese, remove supplier watermark text) → edit description → adjust price (multiply or apply rule) → remove unwanted variants → pick images (remove bad ones) → push to Shopify → choose visibility → wait for result

找到一个商品 → 点导入 → 等加载 → 改标题 → 改描述 → 调价格 → 删变体 → 选图片 → 推送到 Shopify → 选是否上架 → 等结果

一个商品走完这个流程要点十几次。一天上新 20 个商品？几百次点击。每天。
A dozen clicks per product. 20 new products a day? Hundreds of clicks. Every day.

DSers 有 CSV 批量导入和批量推送功能，但那解决的是"量"的问题，不是"判断"的问题。你还是要自己决定每个商品的标题怎么改、价格定多少、哪些图片留哪些删。
DSers has CSV bulk import and bulk push — but that solves the "volume" problem, not the "judgment" problem. You still decide how to rewrite each title, what price to set, which images to keep.

AI agent 能做这件事 / AI agents can do this

AI agent 擅长的恰恰是这类带判断的重复操作。你跟它说：
AI agents are good at exactly this — repetitive tasks that require judgment. You say:

"Import this AliExpress product to my store, translate the title to English, 3x the price, keep only the first 5 images, don't publish to the online store, backend only."

"把这个速卖通商品导入到我的店铺，标题翻译成英文，价格乘 3，只保留前 5 张图，不要上架到在线商店，先放后台。"

它能理解每一个要求。问题是：它没有手。它不能登录 DSers，不能帮你点那十几个按钮。
It understands every requirement. Problem: it has no hands. It can't log into DSers, can't click those buttons for you.

它能做的是调 API。但 DSers 的商品导入链路不是一个 API——是一连串 API，中间有状态依赖、有前置检查、有参数格式的坑。直接把原始 API 暴露给 AI，它会在参数嵌套层级、运费模板绑定这些细节上翻车。
What it can do is call APIs. But DSers' import pipeline isn't one API — it's a chain of APIs with state dependencies, precondition checks, and parameter format traps. Expose the raw APIs to AI and it trips on nested parameter structures, shipping profile bindings, and undocumented prerequisites.

人工智障时刻 / The "artificially incompetent" moment

在做这个之前，我试过直接把 DSers 的原始 API 封装给 AI 用。结果 AI 推送商品到 Shopify 的时候一直报 "shipping profile not found"。
Before building this, I tried wrapping DSers' raw APIs directly for AI. The AI kept getting "shipping profile not found" when pushing products to Shopify.

查了十几次。AI 每次都很认真地构造参数，每次都"看起来"是对的。但就是过不了。
Tried a dozen times. AI carefully constructed parameters each time — everything "looked" right. But it never went through.

最后发现问题：推送接口需要带上 Shopify 的 delivery profile 绑定，但这个参数不在推送 API 的文档里——要先调另一个接口拿到 profile 列表，然后按特定格式拼进去。而且这个格式和 DSers 自己文档里写的还不一样。
Turned out the push endpoint requires Shopify delivery profile bindings — but this parameter isn't in the push API docs. You have to call a separate endpoint first to get the profile list, then assemble it in a specific format. And that format doesn't match what DSers' own documentation says.

AI 不知道有这个前置步骤。它不笨，它只是被丢到了一个没铺好路的环境里。所以我做了这个 MCP：不是把 API 直接扔给 AI，而是把坑都填好，把路铺平，让 AI 只需要走完工作流就行。
AI didn't know about this prerequisite. It's not stupid — it was just dropped into an environment with no paved road. So I built this MCP: instead of throwing raw APIs at AI, I filled every pothole and paved the path, so AI just needs to follow the workflow.
